# Introduction
- 가독성: 모든 팀원이 코드를 쉽게 이해할 수 있어야 합니다.
- 유지보수성: 코드를 쉽게 수정하고 확장할 수 있도록 작성합니다.
- 일관성: 프로젝트 전반에 걸쳐 동일한 스타일을 유지합니다.

# Coding Style
## Guide Line
- 줄 길이: 최대 100자를 권장합니다.
- 들여쓰기: 항상 4칸 공백을 사용합니다.

## Naming Conventions
- 클래스: 카멜 케이스 (CamelCase)를 사용합니다.
- 변수/함수: lowerCamelCase를 사용합니다.
